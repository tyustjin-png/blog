为什么要搭这套系统#

我想要的很简单：一个能替我干活的AI助手，不是聊天玩具。

它应该能帮我管日程、执行定投、推送信息、写文章、做调研——而且是自动的，不需要我每次都去触发。

折腾了大半个月，终于跑通了。记录一下整体架构，也许对同样在折腾的人有点参考价值。

核心组件详解#

1. MBP：纯净终端#

原则：主力笔记本不装任何大模型相关的平台或系统。

MBP只做三件事：

  • VSCode + SSH：远程连Mac Mini写代码、用Claude Code
  • Telegram：跟紫龙和二狗对话
  • 浏览器：看博客、管理Cloudflare

为什么？因为大模型平台装在本地会带来认证、更新、资源占用等一堆问题。把算力和服务全部集中在Mac Mini上，MBP保持轻量干净。

2. Mac Mini M4：一切的引擎#

这是整套系统的核心。7×24小时开机，跑着所有服务：

OpenClaw — AI助手框架

  • 紫龙（主Agent）：用Claude Opus 4.6，处理日常沟通、写作、系统管理
  • 二狗（投资Agent）：专门做投资分析，跟紫龙互不干扰
  • 通过Telegram跟我双向通信

Claude Code — 编程利器

  • 用sub2api生成的API Key认证，不需要OAuth登录
  • 从MBP的VSCode SSH过来直接能用
  • 跟OpenClaw共用同一个API Key，互不冲突

Chrome Debug — 网页数据抓取

  • 端口9222，CDP协议
  • 专用数据目录，跟日常Chrome不冲突
  • launchd开机自启

1Password CLI — 密钥管理

  • Service Account模式，只能访问Agent Vault
  • API Key、密码等敏感信息运行时读取，用完即弃
  • 零明文存储

3. 腾讯云VPS：安全执行器#

定投脚本跑在VPS上，直连币安API。为什么不在Mac Mini上跑？

安全隔离：API Key在VPS本地的1Password里，通过内存读取，不过网络传输。即使Mac Mini被入侵，也拿不到交易密钥。

网络加固：

  • SSH端口改为52222（非默认22）
  • Fail2Ban：3次错误封禁1小时
  • 仅Tailscale内网可连接，公网零端口暴露

4. Tailscale：组网胶水#

Mac Mini和VPS通过Tailscale组成私有网络：

  • Mac Mini：100.99.51.77
  • VPS：100.72.82.26

所有设备间的通信都走Tailscale加密隧道，不暴露公网端口。

5. Cloudflare Pages：博客#

  • Hugo静态站点生成
  • GitHub推送触发自动构建
  • 全球CDN + 自动SSL
  • 域名：jinjin123.com

自动化任务（Cron系统）#

紫龙管理着10个定时任务，每天自动运行：

⏰ 每日时间线

07:00  系统状态重置
07:30  生成当日日程清单（自动从待办池carry over）
07:50  执行BTC定投（VPS直连币安）
08:00  早推送 → Telegram（日程 + 定投结果）
08:10  Web3项目雷达扫描
15:00  午后推送 → Telegram（日程进度）
21:00  晚间总结 → Telegram（日程回顾 + 未完成carry over）

⏰ 周期任务

周一/四 02:00  安全审计 + 系统更新检查
周日 21:30    每周总结

设计原则：

  • 定投只写文件不发消息，由08:00早推送统一发送，避免重复通知
  • 日程推送原样发送，不允许AI模型修改格式
  • 异常才通知，正常静默

安全架构#

五层安全链路：

第1层：Tailscale组网（设备间通信加密）
第2层：SSH密钥认证（禁密码登录）
第3层：1Password零落盘（密钥运行时读取）
第4层：VPS安全加固（非标端口 + Fail2Ban + 零公网暴露）
第5层：操作规则（R012密钥零泄露规则）

核心规则：任何从1Password读取的敏感信息，绝对禁止发送到聊天窗口。 这条规则是用两次泄露事故换来的。

信息流#

外部信息
  ↓
紫龙（AI处理：过滤、分析、总结）
  ↓
Telegram推送给我（只推有价值的）
  ↓
我决策 → 紫龙执行

我不需要主动去查任何东西。定投结果、日程提醒、异常告警——该知道的都会推到Telegram。

本质上，手机从"消费入口"变成了"调度终端"。 我不在手机上刷信息流，而是通过Telegram接收AI系统处理过的高质量信息，在需要时做决策。

踩过的坑#

  1. Discord插件疯狂重连导致Cron重复执行 — 不用的channel必须彻底删除配置，不能只disable
  2. 定投通知重复 — 多个Cron各自发消息，改为统一出口
  3. OAuth登录在SSH场景下失效 — 改用API Key认证彻底解决
  4. Haiku模型自作主张改格式 — 推送prompt必须强调"原样发送，禁止修改"
  5. 密钥泄露到聊天窗口 — 建立R012铁律，从架构层面防止

写在最后#

这套系统的核心思想很简单：把高价值行为交给AI系统自动处理，人只在关键决策点介入。

不是用AI聊天，而是用AI建系统。系统跑起来之后，你会发现：你需要做的事情越来越少，但完成的事情越来越多。
